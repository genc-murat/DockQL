DOL

Docker Observability Language — a domain-specific language and CLI tool for querying, observing, and analyzing Docker infrastructure.

dol 
# List running containers with high CPU
$ dol "observe containers | where cpu > 80% | select name, image, cpu | sort cpu desc"

┌──────────────┬──────────────────┬──────┐
│ name         │ image            │ cpu  │
├──────────────┼──────────────────┼──────┤
│ web-01       │ nginx:1.25       │ 92.3 │
│ api-gateway  │ envoy:1.29       │ 87.1 │
│ worker-03    │ python:3.12      │ 81.5 │

# Stream container crash events
$ dol "events containers where action = die | select time, container, image"

⚡ Features

  • Live container observation with filtering & sorting
  • Real-time Docker event streaming
  • Historical time-travel inspection
  • Alerting with duration guards (print, webhook, restart)
  • Anomaly detection (CPU, memory, restart loops)
  • Persistent SQLite telemetry store
  • Static semantic analysis & type checking
  • Aggregate functions (sum, count, avg, min, max)
  • Arithmetic expressions & string functions
  • If/then/else pipeline branching, case/when
  • Interactive REPL with tab completion
  • Terminal dashboard (top & multi-panel)
  • Export to InfluxDB, Loki, Prometheus
  • ANSI-colored table, CSV, JSON, JSONL output

🚀 Quick Start

terminal 
# Clone and build
$ git clone https://github.com/genc-murat/DockQL.git
$ cd DockQL
$ cargo build --release

# Run a query
$ ./target/release/dol "observe containers"

# Start the interactive REPL
$ ./target/release/dol repl

# Live-updating container monitor
$ ./target/release/dol top

📦 Installation

Pre-built binaries are available for Linux (x86_64, aarch64), macOS (x86_64, aarch64), and Windows (x86_64) via the GitHub Releases page.

Alternatively, build from source with Rust 1.85+:

cargo install dol --git https://github.com/genc-murat/DockQL
